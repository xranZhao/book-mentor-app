## 二、V2 数据层：扩充 data/books.js

### 2.1 现状

每本书的数据结构：
```js
{
  slug, title, author, tags, thesis, concepts, tone,
  quickQuestions, framework, quotes
}
```

### 2.2 V2 需要增加「章节」字段

每本书增加 `chapters` 数组：
```js
chapters: [
  {
    id: "ch01",
    title: "第一章 · 让爱融入生活",
    summary: "本章介绍了非暴力沟通的基本理念……",  // AI 生成摘要
    concepts: [
      { name: "非暴力沟通", definition: "……", example: "……" },
      { name: "NVC四要素", definition: "……", example: "……" }
    ]
  },
  // ... 更多章节
]
```

### 2.3 数据来源

1. **已有**：每本书的 `framework.md` 中已有概念列表和章节引用（如「出现章节: ch01」）
2. **需要补**：从 `chunks/` 原文中提取每章的核心概念 + 定义 + 案例（一页以内摘要）
3. **生成方式**：用 Python 脚本读取 framework.md + chunks → 生成每本书的 chapters JSON

### 2.4 预估体积

- 13 本书 × 平均 8 章 × 每章 ~500 字摘要 = ~52KB（文本，gzip 后 ~15KB）
- 加上概念卡片 ~30KB
- 总计：books.js 从当前 164KB → ~250KB（仍可接受）

### 功能 2: 知识编译 + AI 反馈

**入口**: 概念卡片页底部「✍ 写总结」按钮

**流程**:
1. 用户看了概念后，用自己的话写总结（textarea）
2. 提交后，AI 给出反馈

**System prompt 示例**:
```
你是《非暴力沟通》的作者。用户正在学习第三章「区分观察和评论」。
核心概念是：观察是客观描述事实，评论是给行为贴标签。

用户用自己的话总结了这个概念：「用户写的总结」

请给出简短反馈（3-5句话）：
1. 用户的总结抓住了核心吗？
2. 有没有遗漏或误解的地方？
3. 有没有一个思考题可以让用户再深入一步？
```

**UI**: 复用现有 chat 样式，但 role 改为「学生/导师」

**存储**: 每章的学习记录存在 localStorage：
```js
{
  slug: "nonviolent-communication",
  chapter: "ch03",
  status: "summarized",  // "not_started" | "reading" | "summarized" | "tested" | "mastered"
  summary: "用户写的总结",
  feedback: "AI反馈",
  date: "2026-07-20"
}
```

### 功能 3: 掌握度测试

**入口**: 概念卡片页底部「🧠 开始测试」按钮

**测试形式**: 3 道题，Socratic 式——先出题，用户回答后再给答案和对照

**题型示例**（以《非暴力沟通》第三章为例）:
```
Q1: 以下哪句话是「观察」而非「评论」？
    A. 他昨晚没回我消息
    B. 他根本不在乎我
    C. 他总是这样冷暴力
    D. 他是个不负责的人

Q2: 请用自己的话解释「观察」和「评论」的区别。

Q3: 你今天有没有对某人做过「评论」？
    试着把它改写成「观察」。
```

**评分**: 每题 1-3 分，总分 >= 7/9 算通过 → 章节标记为 "tested"
未通过 → 提示「再花 3 分钟看看概念卡片，明天再来测」

**技术要点**:
- AI 生成每章的测试题（首次打开该章时生成，缓存到 localStorage）
- 用户答案存在 localStorage
- 可通过 `book.chapters[n].concepts` 生成标准化题目（不依赖 AI 也可出题）

## 四、实施顺序建议

按优先级排：

| 序号 | 功能 | 工作量 | 依赖 |
|------|------|--------|------|
| 1 | 章节数据补全（13本书 × N章） | 中 | 无 |
| 2 | 章节概念卡片浏览 | 小 | 1 |
| 3 | 知识编译 + AI 反馈 | 小 | 1, 复用现有 callAI |
| 4 | 掌握度测试 | 中 | 1, 可先用固定模板不依赖 AI |
| 5 | 学习进度仪表盘 | 小 | 2,3,4 |

**建议明天先做 1+2+3**，后天上 4+5。这样第二天就能用上核心学习流程。

## 六、已知问题 & 注意事项

1. **数据质量**: chapters 数据从 framework.md + chunks 提取，最好用 AI 辅助生成每章摘要，手动抽查几本
2. **测试题缓存**: AI 生成的测试题存在 localStorage，不需要每次重新生成（省 token）
3. **离线优先**: 章节卡片、概念卡片数据都在 JS 文件中，不需要网络
4. **向后兼容**: V2 不破坏 V1 的书库/对话功能，只是新增一个 Tab
